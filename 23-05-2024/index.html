Differences Between Commands

DELETE vs DROP vs TRUNCATE:
DELETE: Removes rows from a table based on a condition. It can be rolled back.
DROP: Removes the entire table or database. It cannot be rolled back.
TRUNCATE: Removes all rows from a table without logging individual row deletions. It cannot be rolled back.

ALTER vs UPDATE:
ALTER: Changes the structure of a database object (e.g., adding a column to a table).
UPDATE: Modifies the data within the table without changing the table structure.

COMMIT vs ROLLBACK:
COMMIT: Saves all the changes made during the current transaction to the database. Once committed, the changes cannot be undone.
ROLLBACK: Reverts the changes made during the current transaction. It undoes all operations back to the last commit point, restoring the database to its previous state.

Popular Databases and Their Uses

MySQL: Widely used for web applications, known for its reliability and ease of use.
PostgreSQL: Known for its advanced features and standards compliance, often used in complex applications.
MongoDB: NoSQL database, ideal for handling large volumes of unstructured data.
SQLite: Lightweight, file-based database commonly used in mobile applications.
Oracle: Used for large-scale applications requiring high performance and security.

Storage Engines and Their Explanations

InnoDB (MySQL): Default storage engine in MySQL, supports transactions and foreign keys.
MyISAM (MySQL): Non-transactional storage engine, known for its speed in read-heavy applications.
RocksDB: High-performance, transactional storage engine optimized for fast storage environments.
WiredTiger (MongoDB): Default storage engine in MongoDB, supports compression and concurrency control.
LevelDB: Key-value storage engine developed by Google, optimized for read/write operations.
Berkeley DB: Provides a variety of APIs for developers, supports transactions and replication.
